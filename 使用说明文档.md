# ScrcpyGUI 使用说明文档

本文档面向最终用户与开发者，介绍项目能力、环境准备、功能入口、部署方式与常见问题。

依赖项目：<https://github.com/Genymobile/scrcpy>

---

## 1. 产品简介

ScrcpyGUI 是一个基于 scrcpy 的 Windows 图形界面工具，用于更方便地完成以下工作：

- Android 设备投屏
- USB / WiFi 连接管理
- 常用 scrcpy 参数配置
- 录屏与截图
- 应用管理
- 环境自检与问题诊断

> 说明：历史上的“群控/同步控制”相关代码目前不作为主推荐功能，文档以下内容以当前主界面稳定功能为准。

## 3. 环境要求

### 3.1 系统要求
- Windows 10 / Windows 11
- Python 3.8+

### 3.2 Android 设备要求
- 已开启开发者选项
- 已开启 USB 调试
- 首次连接时已允许 USB 调试授权

### 3.3 scrcpy / adb 要求

程序支持以下几种依赖来源：

1. 用户在程序内手动指定的路径
2. 项目目录 / 打包目录中的本地 scrcpy 环境
3. 系统 PATH 中已安装的 scrcpy / adb
4. 常见安装目录
5. 命令名兜底

默认安装脚本当前会下载 **scrcpy 3.3.4**，但运行时可切换到你指定的其他版本。

## 4. 安装与启动

### 4.1 推荐方式：自动准备环境

```bash
pip install -r requirements.txt
python setup_scrcpy.py
python main.py
```

执行 `setup_scrcpy.py` 后，会在项目目录中下载默认版本的 scrcpy 环境，程序启动时会优先识别本地目录。

### 4.2 手动准备环境

如果你已经有自己的 scrcpy 环境，可以直接：

- 放到项目目录 / 程序目录中
- 或放到系统 PATH 中
- 或在程序菜单中手动指定：
  - ADB 路径
  - scrcpy 路径
  - scrcpy-server 路径

### 4.3 程序启动

```bash
python main.py
```

也可以：

```bash
python main.py --app-manager
```

直接打开应用管理器。

## 5. 依赖路径策略说明

当前运行时依赖解析顺序如下：

**用户显式配置 > 打包内置/本地目录 > PATH > 常见安装目录 > fallback**

这意味着：

- 如果你手动设置了路径，程序优先用你指定的版本。
- 如果没设置，程序会自动找项目目录和系统环境。
- 如果当前目录中的 scrcpy 版本升级了，程序也能识别本地新版。

在“工具”菜单中可使用：

- 设置 ADB 路径
- 设置 scrcpy 路径
- 设置 scrcpy-server 路径
- 恢复自动检测依赖路径

## 6. 使用说明

### 6.1 首次连接设备

1. 在 Android 设备上开启开发者选项
2. 开启 USB 调试
3. 通过 USB 连接电脑
4. 在手机上点击“允许 USB 调试”
5. 打开程序，确认设备状态为“可用”
6. 点击“一键USB连接”开始投屏

### 6.2 WiFi 投屏

1. 先使用 USB 完成一次授权
2. 在程序中点击 WiFi 连接入口
3. 程序会尝试切换到 TCP/IP 模式并连接无线设备
4. 无线连接成功后，可拔掉 USB 线继续使用

### 6.3 常用投屏参数建议

#### 日常清晰模式
- 码率：12 Mbps 左右
- 最大尺寸：1080
- 最大帧率：30

#### 低带宽或远程环境
- 码率：4~8 Mbps
- 最大尺寸：720 或 960
- 最大帧率：20~30

#### 纯录制模式
- 勾选“纯录制模式”
- 选择录制输出路径
- 适合后台录屏，不打开窗口

### 6.4 应用管理器使用

在“工具 -> 应用管理器”中可进行：

- 应用搜索与筛选
- 启动 / 停止 / 卸载
- 清除数据 / 缓存
- 导出 APK
- 查看前台应用与详细信息

## 7. 环境自检与诊断

程序内置两类诊断能力：

### 7.1 环境自检
展示：
- ADB 是否可用
- scrcpy 是否可用
- 版本信息
- 当前设备数量
- 离线 / 未授权设备数量

### 7.2 一键诊断
会输出：
- ADB 路径与来源
- scrcpy 路径与来源
- scrcpy-server 路径与来源
- `adb devices -l`
- `adb version`
- `scrcpy --version`
- 建议处理项

## 8. 常见问题

### 8.1 程序提示找不到 scrcpy 或 ADB

建议按以下顺序检查：

1. 打开“帮助 -> 环境自检”
2. 查看“帮助 -> 一键诊断”中的路径解析
3. 如果路径不对，在“工具”菜单中手动指定正确的 ADB / scrcpy / scrcpy-server
4. 如果本地目录有 scrcpy，确认目录完整，包含：
   - `adb.exe`
   - `scrcpy.exe`
   - `scrcpy-server` 或 `scrcpy-server.jar`

### 8.2 设备显示 unauthorized

- 重新插拔 USB
- 确认手机弹出的授权框已允许
- 必要时执行：

```bash
adb kill-server
adb start-server
```

### 8.3 设备显示 offline

- 检查数据线或 USB 接口
- 检查 WiFi 连接状态
- 对无线设备先执行断开后重连

### 8.4 录屏完成但没自动打开文件

请检查：
- 是否在菜单中勾选“录屏完成后打开目录 / 文件”
- 录屏路径是否有效
- 文件是否已经实际生成

### 8.5 置顶切换没有立即生效

当前版本已做运行时同步处理；若仍偶发失败，可重新勾选一次置顶，或查看日志中是否提示窗口未命中。

## 9. 开发者说明

### 9.1 主要模块

- `main.py`：主界面与主要交互
- `runtime_helpers.py`：依赖路径解析与 scrcpy 命令构建
- `scrcpy_controller.py`：ADB / scrcpy 执行封装
- `config_service.py`：配置与设备参数持久化
- `app_manager.py`：应用管理器
- `wifi_service.py`：WiFi 连接逻辑
- `screenshot_service.py`：截图逻辑
- `build_windows.py`：Windows 打包脚本
- `setup_scrcpy.py`：默认 scrcpy 环境下载脚本

### 9.2 打包说明

#### 打包前准备

```bash
pip install pyinstaller
```

建议在虚拟环境中执行。

#### 常用打包命令

目录模式：

```bash
python build_windows.py --spec ScrcpyGUI_separate.spec
```

单文件模式：

```bash
python build_windows.py --spec ScrcpyGUI_onefile_separate.spec
```

附带 ZIP：

```bash
python build_windows.py --spec ScrcpyGUI_separate.spec --zip
```

#### 当前打包策略

- 打包脚本会自动寻找项目目录下最新的 `scrcpy-win64-v*` 目录
- 如果找到，会将该目录一并打包到发布产物中
- 如果未找到，则发布产物将依赖目标机器上的外部 scrcpy 环境或手动配置路径

## 10. 推荐使用方式总结

如果你想获得最稳的体验，建议使用下面这套方案：

1. 运行 `python setup_scrcpy.py`
2. 启动程序后先看一次“环境自检”
3. 如果你有自己的新版 scrcpy，就在“工具”菜单中手动指定路径
4. 如遇问题，优先看“一键诊断”输出的路径来源与版本信息
